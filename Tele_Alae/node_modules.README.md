# node_modules (explicación)

Este archivo explica por qué la carpeta `node_modules/` suele estar ignorada y cómo manejar dependencias en este proyecto.

Resumen

- `node_modules/` contiene dependencias instaladas desde `npm` y suele ser una carpeta grande y generada automáticamente.
- Por convención, no se sube `node_modules/` al control de versiones; en su lugar se documenta su propósito aquí.

Por qué no versionarla

- Ocupa mucho espacio y puede introducir inconsistencias entre entornos.
- Las dependencias se declaran en los manifiestos (`package.json`, o en este caso, las importaciones `npm:` en `deno.json`) y se pueden regenerar.

Qué hacer en este repo

- No edites archivos dentro de `node_modules/`.
- Para reproducir dependencias, ejecuta las tareas correspondientes o comandos que establezca el proyecto (p. ej. `deno task start` o `npm install` si fuera necesario para herramientas de construcción).

Si necesitas incluir información adicional

- Si por alguna razón necesitas explicar por qué `node_modules/` está en el repositorio en un caso concreto, añade aquí la razón y cualquier instrucción adicional.

Nota

- Colocar este README fuera de la carpeta `node_modules/` permite que el archivo se añada al repositorio sin forzar la inclusión de la carpeta generada.
